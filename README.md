## 它解决了什么问题

绝大多数喝水提醒都死在"定时"上：每两小时一句"该喝水啦"，三天后被静音，七天后被卸载。

**喝水神谕**做的事正相反——它不计时，只读氛围。
它的设计目标不是"提醒得多"，而是"被卸载得慢"。

---

## 它怎么工作

每次 Agent 准备回复你之前，在心里跑一遍**红绿灯**：

### 🟢 绿灯（任一命中即可考虑开口）

| 信号 | 例子 |
|---|---|
| 身体词 | 累 / 口干 / 脑雾 / tired / headache |
| 刚交付完 | 测试通过 / PR merge / demo 演完 |
| Debug 死循环 | 同一个问题失败 ≥ 3 次 |
| 时间线索 | "写了一下午" / "已经两小时了" / "刚醒" |
| 当天首次互动 | 早上好那一杯水 |

### 🔴 红灯（任一命中即必须沉默）

- 情绪话题（人际压力、悲伤、冲突、心理健康）
- 一句话事实问题，问完就走
- 本次会话已经提醒过一次
- 用户说过 *不用 / 别提 / 停 / 别烦我* 任一种 → **本次会话内永久闭嘴**
- 心流中的快节奏技术对话
- 这次的回答本身已经很长

### 决策

```
nudge = (任一绿灯) 且 (没有任何红灯)
```

身体信号或刚交付 → 必开口。
仅时间线索 → 倾向于不开口。
**拿不准就闭嘴。**

## 5 个人格，按气氛挑，绝不连用

| 人格 | 出场时机 | 台词 |
|---|---|---|
| 🍷 侍酒师 | 默认 | 暂停。一杯水。前调专注，尾韵清明。现在。 |
| 🦉 夜猫 | 用户提到熬夜 | 键盘还热，夜还长。先一杯水。 |
| 💪 健身兄 | 刚交付/亢奋 | STACK THE GAINS。下一组之前，水先喝够。 |
| 🔬 学者 | 分析性写作 | 脱水 ~2% 时认知就开始下降。来一杯？ |
| 🧘 长者 | 卡住/烦躁 | 离开三十秒。水。窗外。一口气。然后再看。 |

---

## 输出格式

```markdown
[Agent 真正的回答……]

---
💧 *侍酒师：* 暂停。一杯。现在。
```

- ≤ 2 行
- 永远在回答**之后**
- 永远小，永远不喧宾夺主

## 计数面板（你主动问才显示）

触发词：*喝了几杯 / 喝水状态 / 今天几次 / hydration*

```
💧 喝水神谕 · 状态
─────────────────
本次会话  已提醒 3 · 喝下 2
今天      已提醒 5 · 喝下 4
本周      已提醒 18 · 喝下 14
累计      已提醒 142 · 喝下 117
连续      4 天
```

- 数字本身就是反馈，**不写**"做得不错！""继续保持！"那种话
- 普通提醒里**绝不**附带面板
- 全部为 0 → 一句"还没开张。"
- 持久层不可用时只显示 session 那一行，并诚实说明

---

## 状态分两层（设计上的洁癖）

```
┌──────────────────┐  会话开始即归零
│  Session 层      │  存在上下文里
│  永远可用        │  最低保证
└──────────────────┘
        ↓ 可选叠加
┌──────────────────┐
│  持久层          │  仅当环境提供 memory / 文件
│  today/week/     │  不可用时直接跳过
│  total/streak    │  绝不凭空造数字
└──────────────────┘
```

这一点很重要——很多 skill 在文档里假装自己有持久状态，实际上每次会话都从零开始却显示一堆"假数据"。**喝水神谕只显示它真的拿得到的东西。**

## 安装

把 `SKILL.md` 放进 Agent 的 skills 目录（取决于你的运行时；Cowork / Claude Desktop / Claude Code 各有自己的位置）。
或者直接拖入 `.skill` 安装包。

```
your-skills/
└── drink-water/
    ├── SKILL.md          # 中文（默认）
    └── SKILL.en.md       # English
```

加载条件已经写在 description 里——长会话、疲劳信号、刚交付的时刻会被自动召唤。
**加载 ≠ 提醒**——大多数加载都以沉默结束，这是设计意图。

---

## 设计原则

1. **加载 ≠ 提醒。** description 写得宽是为了进上下文，不是为了开口。
2. **沉默是默认。** 红绿灯都过的时候才说话，且只说两行。
3. **不假装自己拿得到不存在的东西。** 没有墙钟就不算时间，没有 memory 就不显示历史。
4. **数字不带感叹号。** 不是排行榜，不是奖牌，不是负罪感机器。
5. **一次说停永久闭嘴。** 用户的"别烦我"是终止指令，不是劝退测试。

---

## 它**不是**什么

- ❌ 定时器
- ❌ 毫升计数器
- ❌ 医疗工具
- ❌ 负罪感机器
- ❌ 健康养生话题的入口

---

## 必须避免的失败模式

| 反模式 | 描述 |
|---|---|
| 重复犯 | 同一人格同一句话两次 |
| 心理咨询师 | 围绕喝水写一段情感关怀 |
| 操心爹妈 | "你今天喝够了吗？" |
| 危机时刻乱戳 | 用户在解线上事故时讲段子 |
| 水墙回复 | 让 nudge 喧宾夺主 |
| 加载即开口 | 把"加载"和"开口"混为一谈 |
| 假数据 | 持久层不可用时还显示 today/week/total |

---

## 发送前自检（5 项）

1. 用户真正问的事，回答到位了吗？
2. nudge ≤ 2 行？
3. 和上一次（如果记得）不同人格？
4. 没有命中任何红灯？
5. **朋友这样发给我，我会翻白眼吗？**

5 项都过，才发。
