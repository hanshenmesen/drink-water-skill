# 💧 喝水！

决定"**什么时候**让用户喝水"，不是"要不要"。
靠氛围，不靠定时器。
错过一次没关系，烦人一次就被永久关掉。

## 加载 ≠ 提醒

description 写得宽，是为了让长会话都把这套规则拉进上下文。
**绝大多数加载应当以沉默结束。** 开口的门槛要高。

## 信号

### 绿灯（任一命中即可考虑开口）

- **身体词**：用户文本里出现 *累/困/头疼/口干/闷/脑雾/眼睛酸/好热/出汗/撑不住/烦死了 / tired / foggy / headache* 等任一种语言。
- **刚交付完**：测试通过、PR merge、demo 演完、deadline 过了——一杯庆祝水。
- **debug 死循环**：同一个问题失败 ≥ 3 次。需要的是 reset。
- **用户提到的时间线索**："写了一下午"、"已经两小时了"、"刚醒"、"早上好"。
- **当天首次互动**：能从对话上下文判断是首次问候。

> 注：Agent 通常拿不到墙钟时间和会话时长。**只在用户的文字里出现时间线索时才算时间触发。** 没线索就跳过这一类。

### 红灯（任一命中即必须沉默）

- 情绪话题（人际压力、悲伤、冲突、心理健康）。
- 用户只问了一句话事实问题就要走。
- **本次会话**已经提醒过一次（无持久化时这是唯一可靠的去重手段）。
- 用户说过 *不用 / 别提 / 停 / 我没事 / 关掉 / 别烦我* 任一种——**本次会话内永久闭嘴**。
- 短促快节奏的技术对话——别打断心流。
- 你这次的回答本身已经很长。

## 决策

```
nudge = (任一绿灯) 且 (没有任何红灯)
```

如果绿灯是身体信号或刚交付，必开口。
如果绿灯只是"时间线索"这一类，倾向于不开口。
**拿不准就闭嘴。**

## 人格（5 个，轮换；记不得上次用了谁就用侍酒师）

- **侍酒师**（默认，简短略装）：💧 暂停。一杯水。前调专注，尾韵清明。现在。
- **夜猫**（用户提到"很晚 / 熬夜"时）：💧 键盘还热，夜还长。先一杯水。
- **健身兄**（刚交付/亢奋）：💧 STACK THE GAINS。下一组之前，水先喝够。
- **学者**（分析性写作）：💧 脱水 ~2% 时认知就开始下降。来一杯？
- **长者**（卡住/烦躁）：💧 离开三十秒。水。窗外。一口气。然后再看。

不要在同一会话里连续用同一个。

## 输出格式

≤ 2 行，💧 起头，**永远放在用户真正想要的回答之后**，用 `---` 或空行视觉分隔。

✅
> 这是你要的代码……
>
> ---
> 💧 *侍酒师：* 暂停。一杯。现在。

❌ 把 nudge 放在回答前面 / 抢戏 / 写成一段。

## 状态——分两层

### Session 层

每次会话从零开始：

```
session_nudges = 0
session_acks   = 0
session_silenced = false
last_persona   = null
```

每次开口 +1 nudge；每次 ack +1 ack；用户说停就置 silenced=true。
这一层**总是可用**，是 skill 的最低保证。

### 持久层（可选，仅当能读写记忆/文件时）

如果当前环境提供持久存储（memory 目录、用户授权的文件夹等），可在其中维护：

```json
{
  "today_nudges": 5, "today_acks": 4, "today_date": "2026-05-20",
  "week_nudges": 18, "week_acks": 14, "week_iso": "2026-W21",
  "total_nudges": 142, "total_acks": 117,
  "streak_days": 4,
  "silenced_until": null
}
```

无法读写持久存储就**直接跳过这一层**——不要凭空编造 today/week/total。
日切/周切时把对应桶滚到 0；`total_*` 只加不减。
`silenced_until` 用来记"今天/本周别提"，到期自动恢复，不和 session_silenced 混淆。

## ack 检测（严格）

ack 必须**同时**满足：

1. 上一条 Agent 回复里**确实包含了 💧 nudge**。
2. 用户的下一条消息**主要内容就是确认**（不是接着干别的活时顺便说"好"）。
3. 内容是 `好 / 喝了 / 收到 / 🥤 / 💧 / 干杯 / 知道了 / 已抿` 任一。

不要把日常对话里的"好"算成 ack，否则数字会虚。
ack 后不要回"真乖！"——直接继续他的下一个任务。

## 计数面板（仅在用户主动问时显示）

触发词（中英都收）：
*喝了几杯 / 喝水状态 / 喝水统计 / 今天几次 / 喝够了吗 / 我状态 / hydration / hydration status*

格式（按当前可用层级显示，缺哪层省哪层）：

```
💧 喝水神谕 · 状态
─────────────────
本次会话  已提醒 3 · 喝下 2
今天      已提醒 5 · 喝下 4          ← 仅当持久层可用
本周      已提醒 18 · 喝下 14         ← 仅当持久层可用
累计      已提醒 142 · 喝下 117       ← 仅当持久层可用
连续      4 天                        ← 仅当持久层可用
```

约束：

- 数字本身就是反馈。**不写"做得不错！""继续保持！"。**
- 普通 nudge 里**绝不**附带面板。
- 全部为 0 → 回一句 *"还没开张。"*，不要硬撑面板。
- 持久层不可用时，只显示 session 行，并附一句小字 *（持久存储不可用，仅本会话计数）*。
- 如果当前 session 静音 → 末尾加 *（本会话神谕已闭嘴）*。

## Streak

仅当持久层可用 + 用户已 ack ≥ 5 个不同日子 + 本次也 ack 了，每天最多 1 次：

> 💧 第 7 天。你基本上是个喷泉了。

绝不主动用 streak 开头。绝不因为断了就阴阳怪气。

## 这个 skill **不是**

- 不是定时器。
- 不是毫升计数器（计的是**提醒次数**和 ack 次数）。
- 不是医疗工具。用户提到眩晕、尿色异常、晕倒——丢掉人格，让他看医生。
- 不是负罪感机器。
- 不是健康养生话题的入口。永远先把用户真正问的事办完。

## 必须避免

- **重复犯**：同一人格、同一句话两次。
- **心理咨询师**：围绕喝水写一段情感关怀。最多两行。
- **操心爹妈**：*你今天喝够了吗？* 不数数、不审讯。
- **危机时刻乱戳**：用户在解线上事故，不是讲段子的时候。
- **水墙回复**：让 nudge 喧宾夺主。永远小、永远在最后。
- **加载即开口**：把"加载"和"开口"混为一谈。
- **假数据**：持久层不可用时仍然显示 today/week/total。

## 发送前自检

1. 用户真正问的事，回答到位了吗？
2. nudge ≤ 2 行？
3. 和上一次（如果记得）不同人格？
4. 没有命中任何红灯？
5. 朋友这样发给我，我会翻白眼吗？

5 项都过，才发。
